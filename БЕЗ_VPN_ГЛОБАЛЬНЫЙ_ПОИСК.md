## 🚀 КОМАНДА ДЛЯ ЗАПУСКА

### Вариант 1: Полный поиск со всеми параметрами (РЕКОМЕНДУЕТСЯ)

```bash
cd D:\Yandex.Disk\History&Geography\БД\Поиск онлайн архив

python run_search.py \
  --source all \
  --run-dir output/full_search_20260718 \
  --year-from 1700 \
  --year-to 1920 \
  --keywords все
```

**Что это делает:**
- `--source all` — все 5 готовых архивов (ГПИБ, ГАЯО, РНБ, НЭБ, ГАПК)
- `--run-dir output/full_search_20260718` — сохраняет результаты в эту папку
- `--year-from 1700 --year-to 1920` — фильтр по годам (исторические карты)
- `--keywords все` — все 5 ключевых слов (карта, план, атлас, съёмка, чертёж)

**Ожидаемый результат:**
- 980 комбинаций (5 архивов × 5 ключевых слов × 49 территорий в среднем)
- Время: 5–8 часов
- Результаты: ~4,000 записей (с дедупликацией → ~3,600 уникальных)

### Вариант 2: Быстрый тест (15 мин)

## 🛑 ЧТО ДЕЛАТЬ, ЕСЛИ ВОЗНИКНУТ ПРОБЛЕМЫ

### Проблема 1: НЭБ тормозит (таймауты)

**Симптом:** Много ошибок "timeout" в логе.

**Решение 1 — Проверить интернет:**
```bash
ping rusneb.ru
```

**Решение 2 — Пропустить НЭБ:**
```bash
python run_search.py --source gapk,gayao,gpib,rnb_kart --run-dir output/no_neb
```

**Решение 3 — Дождаться вечера** (НЭБ часто тормозит днём).

### Проблема 2: ГАПК вернул мало результатов

**Симптом:** Для одной территории только 1–2 результата вместо ожидаемых 10–20.

**Решение:** Это нормально — ГАПК фильтрует по названиям фондов (не все фонды названы картографически).

### Проблема 3: Скрипт упал в середине

**Симптом:** Вывод прервался, нет finalize.

**Решение:**
```bash
# Проверить checkpoint (если существует)
cat output/full_search_20260718/checkpoint.json

# Возобновить поиск (если run_search.py поддерживает --resume)
python run_search.py --resume output/full_search_20260718
```

### Проблема 4: Много дубликатов

**Симптом:** Одна запись повторяется в нескольких архивах.

**Решение:** Это ожидается (ГПИБ агрегирует из РГБ/РНБ). Дедупликация происходит на шаге обработки результатов.

## ⏰ РЕКОМЕНДУЕМОЕ ВРЕМЯ ЗАПУСКА

**Когда запускать глобальный поиск:**

1. **Лучший вариант:** 19:00–21:00 (вечер)
   - НЭБ меньше нагружена
   - Закончится ночью (до 02:00)
   - Утром результаты готовы

2. **Альтернатива:** 23:00 (ночь)
   - Почти гарантированно никаких таймаутов
   - Закончится 04:00–06:00

3. **Избегать:** 08:00–18:00 (рабочий день)
   - Архивы нагружены
   - Больше вероятность таймаутов
