# 📞 ЗАПРОС ДЛЯ ДИРЕКТОРОВ: Техническая информация

Письмо для директоров Росархива и РГАДА с просьбой о технической информации для интеграции в систему автоматизированного поиска.

## Росархив (Российский государственный архив)

**Контакт:** https://www.rusarchives.ru (отдел IT / информационных технологий)

### Запрос (техническая информация):

1. **Статус API**
   - Есть ли открытый REST API для поиска по фондам?
   - Если есть — какова документация и endpoint'ы?
   - Если нет — планируется ли его разработка?

2. **Поиск по картографии**
   - Как структурирован каталог? (иерархия: Фонд → Опись → Дело)
   - Есть ли фильтр по типу документа (карты, планы, чертежи)?
   - Какие коды используются для картографических материалов?

3. **Автоматизация**
   - Разрешен ли автоматизированный скрепинг?
   - Если есть API — какие ограничения по rate limiting?
   - Требуется ли регистрация / ключ доступа?

4. **Историческое охватывание**
   - Какие временные периоды охватывает каталог?
   - Есть ли карты XVIII–XIX веков по Калужской, Пермской, Смоленской, Ярославской регионам?

5. **Форма ответа**
   - Как структурирована информация о документе? (JSON, XML, HTML)
   - Какие поля обязательны? (название, дата, шифр, ссылка)

**Образец письма:**

```
Уважаемые коллеги!

Мы разрабатываем систему автоматизированного поиска картографических материалов

(ГПИБ, ГАЯО, РНБ, НЭБ, ГАПК) и собрала ~3,600 уникальных записей.

Для расширения охвата нам нужна техническая информация:

1. Работает ли в настоящий момент публичный API Росархива?
2. Если да — какова структура запроса для поиска по типу документа (карты)?
3. Какие ограничения по частоте запросов?
4. Требуется ли регистрация для автоматизированного доступа?

## РГАДА (Российский государственный архив древних актов)

**Контакт:** https://www.rgada.info (отдел информационных технологий)

### Запрос (техническая информация):

1. **Каталог онлайн**
   - Есть ли открытая база данных фондов в интернете?
   - URL каталога (если не https://www.rgada.info/main/)
   - Есть ли API для автоматизированного поиска?

2. **Картографические материалы**
   - Какие фонды содержат карты, планы, чертежи XVI–XVIII века?
   - Есть ли в каталоге фильтр по типу документа?
   - Коды или ключевые слова для поиска картографии?

3. **Структура данных**
   - Иерархия: Фонд → Опись → Дело → Единица хранения?
   - Какие поля доступны для каждой записи?
   - Есть ли прямые ссылки на оцифрованные копии?

4. **Поиск и фильтрация**
   - Как устроена форма поиска на сайте? (параметры URL)
   - Можно ли искать по территориям (Калужская, Пермская, Смоленская, Ярославская)?
   - Есть ли фильтры по датам?

5. **Автоматизация**
   - Разрешена ли автоматизация (скрепинг / API-запросы)?
   - Требуется ли регистрация?
   - Какие ограничения по частоте запросов (если есть)?

6. **Форматы данных**
   - Какой формат ответа использует каталог? (HTML, JSON, XML?)
   - Возможно ли экспортировать результаты поиска?

```

## Рекомендации по переписке

### Что включить в письмо:

1. **Представление:** Кто вы, чем занимаетесь
2. **Контекст:** Зачем нужна информация (исторический проект, система поиска)
3. **Конкретика:** Ровно 3–5 вопросов (не более)
4. **Форма ответа:** "Ответить на письмо / отправить PDF с документацией / назначить встречу"
5. **Сроки:** "Спасибо, нам нужно к [дата]"

### Когда ожидать ответ:

- **Быстрый ответ (1–3 дня):** Если вопрос техничный и есть контакт IT-отдела
- **Средний ответ (1–2 недели):** Если нужно согласование с руководством
- **Медленный ответ (месяц+):** Если архив консервативен или перегружен

### План B (если ответа нет):

1. Использовать DevTools на сайте архива → перехватить HTTP запросы
2. Запустить Selenium/Playwright для автоматизации поиска
3. Обратиться к коллегам в других архивах (может быть знают внутренние API)
