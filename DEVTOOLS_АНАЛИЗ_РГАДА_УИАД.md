# DevTools анализ: РГАДА и ГИС УИАД

Пошаговая инструкция для выявления параметров поиска на обоих сайтах.

---

## 1️⃣ РГАДА — http://rgada.info/poisk/index.php

### Шаг 1: Открыть сайт в Chrome
```
1. Перейти на http://rgada.info/poisk/index.php
2. Открыть DevTools: F12 → вкладка "Network"
3. Очистить логи: Ctrl+L
```

### Шаг 2: Выполнить поиск
```
1. В форме ввести: "карта" (или "план", "чертёж")
2. Нажать кнопку "Найти"
3. В DevTools Network смотреть XHR запрос
```

### Шаг 3: Анализ запроса
```
Вопросы:
- Какой URL запроса? (GET или POST?)
- Какие параметры передаются? (q=?, fund=?, ...)
- Какой метод? (GET /poisk/index.php?... или POST?)
- Кодировка ответа? (посмотреть в Response Headers → Content-Type)
```

### Шаг 4: Анализ результатов
```
1. Посмотреть Response в DevTools
2. Это HTML или JSON?
3. Если HTML:
   - Инспектировать элемент карточки результата
   - Найти CSS class/id для селектора
   - Примеры селекторов: div.search-result, li.item, tr.row
4. Если JSON:
   - Посмотреть структуру в Response tab
   - Найти ключи: title, shelfmark, year, url
```

### Шаг 5: Проверить пагинацию
```
1. Есть ли кнопка "Следующая страница"?
2. Если есть, нажать и посмотреть новый запрос
3. Изменился ли URL или параметры? (page=?, offset=?, ...)
```

### Шаг 6: Сохранить результаты
```
Скопировать из DevTools:
1. URL запроса (полностью)
2. Параметры (query string)
3. Примеры из Response (first 3 items)
4. Сохранить в файл: DEVTOOLS_RGADA_РЕЗУЛЬТАТЫ.txt
```

## 2️⃣ ГИС УИАД — https://online.archives.ru/search/full/

### Шаг 1: Открыть сайт в Chrome
```
1. Перейти на https://online.archives.ru/search/full/
2. Открыть DevTools: F12 → вкладка "Network"
3. Очистить логи: Ctrl+L
```

### Шаг 2: Выполнить поиск
```
1. В поле поиска ввести: "карта" (или "план")
2. Нажать Enter или кнопку поиска
3. В DevTools смотреть ВСЕ запросы (может быть несколько XHR)
```

### Шаг 3: Анализ запроса (может быть сложнее!)
```
Вопросы:
- Есть ли XHR запрос? (может быть AJAX)
- Если XHR: какой URL и параметры?
- Если обычный GET: просто измениться URL страницы
- Content-Type ответа? (application/json или text/html?)
```

### Шаг 4: Анализ результатов
```
1. Если JSON API:
   - Response будет JSON
   - Найти ключи: id, title, shelfmark, archive_name, year
   - Может быть вложенная структура (items → [{ ... }])
2. Если HTML:
   - Инспектировать карточку результата
   - Найти селекторы (class, id, tag)
```

### Шаг 5: Проверить пагинацию
```
1. Есть ли кнопка "Загрузить ещё"?
2. Или "Следующая"?
3. При клике: новый XHR запрос?
4. Какие параметры меняются? (page, offset, limit, ...)
```

### Шаг 6: Сохранить результаты
```
Скопировать из DevTools:
1. URL запроса (полностью)
2. Method (GET/POST)
3. Request Headers (важно: есть ли Authorization, Cookie, ...)
4. Request Body (если POST)
5. Примеры из Response (first 3 items)
6. Сохранить в файл: DEVTOOLS_UIAD_РЕЗУЛЬТАТЫ.txt
```

## 📋 Checklist анализа

### Для каждого архива:

- [ ] Форма поиска работает (вводится текст, кнопка нажимается)
- [ ] DevTools перехватывает запрос (есть XHR в Network)
- [ ] Определён URL запроса (скопирован полностью)
- [ ] Определены параметры (q=?, page=?, ...)
- [ ] Определён тип ответа (HTML или JSON)
- [ ] Найдены селекторы результатов (для HTML) или структура (для JSON)
- [ ] Найдены поля: название, шифр, год, URL
- [ ] Проверена пагинация (есть ли ещё страницы)
- [ ] Результаты сохранены в файл
